# FlexKV ROCm GPU Backend

This directory hosts the ROCm/HIP compilation artifacts for FlexKV.

## Layout

| Path | Purpose |
| ---- | ------- |
| `.hipified/` | Auto-generated by `hipify-perl` from `csrc/gpu_backend/nvidia/*.cu*`. Regenerated on every build; **never** committed. |
| `.gitignore` | Excludes `.hipified/` from version control. |

## Build flow

1. `FLEXKV_GPU_BACKEND=rocm pip install -e .`
2. `setup.py` selects [`build_backends/rocm_builder.py`](../../../build_backends/rocm_builder.py).
3. `RocmBuilder.configure_env()`:
   - Sets `PYTORCH_ROCM_ARCH=gfx90a;gfx942` if not already set.
   - Runs `hipify-perl` on
     `csrc/gpu_backend/nvidia/{transfer.cu,transfer.cuh,gtensor_handler.cuh}`
     → `csrc/gpu_backend/rocm/.hipified/`.
4. ROCm PyTorch's `CUDAExtension` automatically dispatches to `hipcc`.
5. The CPU thread-group source
   [`csrc/gpu_backend/nvidia/tp_transfer_thread_group.cpp`](../nvidia/tp_transfer_thread_group.cpp)
   is **shared** with the NVIDIA build because it uses the
   cross-vendor `gpu_*` macros from
   [`csrc/gpu_backend/gpu_types.h`](../gpu_types.h).

## What is NOT here

- No `gds/` subdirectory — ROCm has no cuFile equivalent today.
  Direct-storage workloads fall back to the GPU-agnostic POSIX/io_uring
  path in [`csrc/transfer_ssd.cpp`](../../transfer_ssd.cpp).

## Requirements

- ROCm 5.7+ (recommended 6.x) on host
- PyTorch built against ROCm
- `hipify-perl` on `PATH` (typically `/opt/rocm/bin/hipify-perl`)

## Manual smoke test

```bash
hipify-perl --version
FLEXKV_GPU_BACKEND=rocm python -c "from flexkv.gpu_backend import current_backend; print(current_backend, current_backend.is_available())"
```
